# KAMVARA — Premium Plant Protein E-Commerce

A premium, full-stack e-commerce website for **KAMVARA**, a plant-based protein supplement brand. Built with a dark, athletic aesthetic featuring gold accents, botanical textures, and smooth animations.

## Tech Stack

- **Framework:** [TanStack Start](https://tanstack.com/start) (React 19 + Vite 7)
- **Styling:** Tailwind CSS v4
- **Routing:** File-based routing via `@tanstack/react-router`
- **State Management:** React hooks + custom store (cart)
- **UI Components:** Radix UI primitives + custom components
- **Notifications:** Sonner toast library
- **Icons:** Lucide React

## Features

- Product catalog with 6 flavors (Orange Cream, Cold Brew Coffee, Double Chocolate, Strawberry Burst, Vanilla Ice Cream, Unflavored Raw)
- Animated hero section with product showcase
- Add to cart with localStorage persistence
- Cart page with quantity management
- About page with brand story and manufacturing process
- Responsive design for mobile & desktop
- Premium dark theme with gold accents

## Prerequisites

- [Node.js](https://nodejs.org/) 20+ (recommended)
- [Bun](https://bun.sh/) (optional but faster — recommended)

## Getting Started

### 1. Clone the Repository

```bash
git clone <your-repo-url>
cd <project-folder>
```

### 2. Install Dependencies

Using Bun (recommended):
```bash
bun install
```

Or using npm:
```bash
npm install
```

### 3. Run the Development Server

```bash
bun run dev
```

The app will be available at **http://localhost:8080**

### 4. Build for Production

```bash
bun run build
```

### 5. Preview Production Build

```bash
bun run preview
```

│   │   └── site-nav.tsx

├── tsconfig.json

> **Note:** `src/routeTree.gen.ts` is auto-generated by TanStack Router. Do **not** edit it manually.

## Customization

- **Products:** Edit `src/lib/products.ts` to add/remove products or change pricing
- **Theme:** Modify colors in `src/styles.css` under `@theme`
- **Images:** Replace files in `src/assets/` with your own product photography

## Important Notes

- Cart data is stored in **localStorage** (persists across page reloads)
- No backend/database is currently connected — all product data is static
- For full-stack features (auth, database, payments), connect [Lovable Cloud](https://docs.lovable.dev/features/cloud)
